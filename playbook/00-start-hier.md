# Start hier — Ideen auf Probe

So startest du einen neuen 6-Monats-Lauf — unabhängig von der konkreten Idee.

---

## Voraussetzungen

Bevor du einen Cycle startest, musst du haben:

- [ ] **Eine B2B-Idee** — kein Idea Sourcing in diesem Playbook; die Idee steht am Day 0 fest
- [ ] **~5 Stunden pro Woche** — Spikes in Monat 3 und 4 sind willkommen
- [ ] **DACH-Markt** — Käufer auf LinkedIn erreichbar
- [ ] **Tools:** Sales Navigator, Compleadly (nur Erstkontakt), Notion (privat), GitHub (öffentlich)

---

## Day 0 — Cycle-Start (ca. 2 Stunden)

### 1. Codename vergeben

Jeder Lauf bekommt einen **Codename** — kein Produktname, keine Marke.

| Gut | Schlecht |
|-----|----------|
| Projekt Linde | SaaS-Idee-2026 |
| Projekt Ufer | AI-Tool für HR |
| Projekt Kasten | Mein Startup |

**Warum:** Du kannst öffentlich dokumentieren (*„Monat 3 bei Projekt Linde"*) ohne die Idee zu verraten (L3).

## Offenlegungsrichtlinien

Alles *kann* öffentlich werden. Standard-Tiefe:

| Artefakt | Standard-Tiefe | Anpassung |
|----------|----------------|-----------|
| Playbook-Schritte | L1 — voll | — |
| Interview-Anzahl | L2 — gerundet/aggregiert | Case-by-case |
| Muster & Learnings | L2 — ohne Namen | Case-by-case |
| Produktidee | L3 — Kategorie, nicht Name | Case-by-case |
| Pricing | L3 — Range, nicht exakt | Case-by-case |
| Stop/Weitermachen | L4 — Entscheidung + 3–5 Bullets | Meist voll |

**Faustregel pro Post:** Erkenntnis öffentlich, Rohdaten in Notion.

## Stop oder Weitermachen — Kurzdefinition

| | **Stop** | **Weitermachen** |
|---|----------|------------|
| Bedeutung | Idee stoppen, sauber archivieren | Idee weiterverfolgen |
| Erfolg? | **Ja** — 200+ Stunden gespart | Mandat für Phase 2 |
| Öffentlich | Lessons + Playbook-Version | Nächster Schritt + Überzeugungsgründe |
| Phase 2 | — | POC → Prototyp → MVP → erste zahlende Kunden |
