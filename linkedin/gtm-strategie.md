# LinkedIn GTM-Strategie

Deutsch. DACH. B2B. ~5 Stunden/Woche. Öffentliches Playbook ab Tag eins.

---

## Architektur

```
GitHub (Source of Truth) → Notion (hübsch lesen) → LinkedIn (Distribution + Validierung)
         ↑                                              ↓
    Playbook v0.5                                  Discovery-Gespräche
```

**LinkedIn ist zwei Dinge:**
1. **Bühne** — Playbook-Fortschritt, Methodik, Kill/Hustle
2. **Labor** — ICP finden, Gespräche buchen, Sprache testen

---

## Profil-Setup

### Headline (anpassen wenn Name steht)

> [Playbook-Name] — 6 Monate B2B-Validierung in DACH, öffentlich dokumentiert

### Info-Bereich (Struktur)

1. **Was ich tue** — Eine Idee, 6 Monate, Kill or Hustle
2. **Für wen** — B2B-Gründer in DACH (auch wenn primär für dich)
3. **Was du bekommst** — Playbook, Newsletter, ehrliche Logs
4. **Was ich nicht bin** — Kein VC, kein YC, kein „Growth Hacking"
5. **Links** — Notion (lesen), GitHub (Versionen)

### Featured (3 Slots)

| Slot | Inhalt |
|------|--------|
| 1 | Notion-Playbook (Reading Layer) |
| 2 | GitHub-Repo (Changelog / Version) |
| 3 | Newsletter (nach Launch) |

**Creator Mode:** An.

## Was du nicht tun solltest

- Company Page als Hauptkanal
- Produkt pitchen vor Gate G3
- Englische Posts (erst nach stabiler deutscher Kadenz)
- Jede Interview-Notiz veröffentlichen
- Compleadly für Follow-ups automatisieren

| 2–4 | 2×/Woche posten (Methodik + Playbook-Ankündigung) |
| 5–6 | Newsletter erstellen, Ausgabe #0 |
| 7–8 | Newsletter-Launch + Featured aktualisieren |
| 9–24 | Playbook-Log parallel zum aktiven Cycle |

## Sync GitHub ↔ Notion

Bei jedem Release (v1.0, v1.1, …):

1. GitHub finalisieren + taggen
2. Geänderte Abschnitte nach Notion kopieren (~30 Min.)
3. Footer: *„Version X.Y — [GitHub](link)"*
4. LinkedIn-Post: Was sich geändert hat

**Nicht** täglich syncen — nur bei Version-Bumps.
